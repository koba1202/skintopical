# ステロイド塗布管理アプリ — 要件定義書

## 1. プロジェクト概要

### 1.1 背景・課題

アトピー性皮膚炎の治療においてステロイド外用薬は中心的な役割を果たすが、多くの患者が以下の理由で治療を完遂できず、症状のぶり返し（リバウンド）を繰り返している。

| # | 課題 | 具体的な行動 |
|---|------|-------------|
| 1 | 途中離脱 | 症状が改善すると自己判断で塗布を中止する |
| 2 | 塗布量不足 | 「もったいない」意識から薄く塗り、十分な薬効が得られない |
| 3 | テーパリング失敗 | 漸減スケジュール（1日2回→1回→隔日→週2回…）を自力で管理できない |

### 1.2 アプリのミッション

> **「医師の指示に基づく治療計画を、最後まで正しくやり遂げるための伴走者」**

アプリ自体が治療方針を決定するものではなく、ユーザーが医師から受けた指示を忠実に実行・継続できるよう支援するツールである。

### 1.3 ターゲットユーザー

- アトピー性皮膚炎でステロイド外用薬を処方されている患者（成人）
- 症状の改善→自己中断→再燃を繰り返した経験がある方
- スマートフォンを日常的に使用している方

### 1.4 技術方針

| 項目 | 方針 |
|------|------|
| フェーズ | プロトタイプ（MVP） |
| プラットフォーム | Webアプリ（React） |
| データ永続化 | ローカルストレージ or IndexedDB（MVP段階） |
| 将来展望 | モバイルアプリ化（React Native等） |

## 2. 機能要件

### 2.1 塗布スケジュール＆リマインド【最重要】

治療計画の登録・表示・通知を行い、途中離脱を防ぐ中核機能。

#### 2.1.1 治療フェーズの登録

ユーザーが医師の指示に基づき、治療を複数の「フェーズ」として登録する。

```
例：医師の指示が「まず1日2回を2週間、次に1日1回を2週間、その後隔日で2週間」の場合

  フェーズ1: 1日2回（朝・夜）— 14日間
  フェーズ2: 1日1回（夜）    — 14日間
  フェーズ3: 隔日（夜）      — 14日間
  フェーズ4: 週2回（夜）     — 14日間（保湿への移行期）
```

各フェーズに設定する項目：

- フェーズ名（自由入力 or プリセット選択）
- 塗布頻度（1日2回 / 1日1回 / 隔日 / 週N回 / カスタム）
- 塗布タイミング（朝 / 夜 / 朝夜 / カスタム時刻）
- 期間（日数 or 「症状を見て判断」フラグ）
- 対象部位（後述の身体部位マッピングと連動）
- 使用する薬剤（薬の名称・ランク）

#### 2.1.2 今日のタスク表示

アプリのホーム画面に「今日やるべきこと」を明確に表示する。

- 現在のフェーズとその進捗（例：フェーズ2 — 8日目/14日間）
- 今日の塗布回数・タイミング
- 対象部位と使用する薬
- 「塗った」ボタン（ワンタップ記録）

#### 2.1.3 リマインド通知

- 設定した塗布タイミングに合わせたプッシュ通知（※MVP段階ではブラウザ通知）
- 塗布記録がない場合のフォローアップ通知（30分後に再通知）
- フェーズ切り替え日の事前通知（「明日から1日1回に減らします」）

#### 2.1.4 テーパリング進捗の可視化

- 全フェーズの進捗をプログレスバーで表示
- 「寛解」というゴールまでの距離を視覚化
- 達成したフェーズにはチェックマークを付与

---

### 2.2 身体部位マッピング

塗布する部位を視覚的に管理する機能。

#### 2.2.1 ボディマップUI

- 人体の前面・背面のイラスト（SVGベース）
- タップ/クリックで部位を選択可能
- 主要部位の区分：
  - 顔（額・頬・顎・首）
  - 上肢（上腕・前腕・手・肘の内側/外側）
  - 体幹（胸・腹・背中）
  - 下肢（大腿・下腿・足・膝の裏）

#### 2.2.2 部位ごとの設定

各部位に対して以下を紐付ける：

- 使用する薬剤名（例：リンデロンV軟膏）
- ステロイドのランク（Strongest / Very Strong / Strong / Medium / Weak）
- 症状の重症度（なし / 軽度 / 中等度 / 重度）※任意入力
- FTUの目安値（自動算出 — 後述）

#### 2.2.3 ボディマップの色分け表示

- 薬剤のランク別に色分け（例：赤系→Strong / 黄系→Medium）
- 塗布済み/未塗布の状態表示（グレーアウト / ハイライト）

---

### 2.3 FTU（フィンガーチップユニット）ガイド

塗布量の「節約しすぎ」を防ぐ教育・ガイド機能。

#### 2.3.1 FTUとは

- 大人の人差し指の先端から第一関節までチューブから絞り出した量 = 1FTU ≒ 約0.5g
- 1FTUで塗れる面積 ≒ 手のひら2枚分

#### 2.3.2 部位別FTU目安表示

ボディマップと連動し、選択した部位に必要なFTUを表示する。

| 部位（成人） | FTU目安 |
|-------------|--------|
| 顔＋首 | 2.5 FTU |
| 片方の上肢全体 | 4 FTU |
| 片方の手（両面） | 1 FTU |
| 体幹（前面） | 7 FTU |
| 体幹（背面） | 7 FTU |
| 片方の下肢全体 | 8 FTU |
| 片方の足 | 2 FTU |

※上記は一般的な成人の目安値。将来的には体格入力による調整も検討。

#### 2.3.3 視覚的ガイド

- 指先にチューブから薬を出すイラスト/アニメーション
- 「ティッシュに薬を取って貼り付けたとき、ティッシュが透けるくらい」の目安解説
- 塗布量チェック：部位選択 → 「この部位にはN FTU（約N.Ng）必要です」の表示

#### 2.3.4 チューブ残量管理（将来機能）

- チューブの容量（5g / 10g / 20gなど）を登録
- 使用量から残量を推定し、次回受診前の補充を促す

```
[1] ホーム画面
```

## 4. データモデル（MVP）

### 4.1 エンティティ定義

```
TreatmentPlan（治療計画）
├── id: string (UUID)
├── name: string（計画名。例：「2024年冬の治療」）
├── startDate: Date
├── status: "active" | "completed" | "paused"
├── createdAt: Date
└── phases: Phase[]

Phase（フェーズ）
├── id: string (UUID)
├── order: number（フェーズの順番）
├── name: string（例：「集中治療期」「漸減期」）
├── frequency: FrequencyType
│   └── "twice_daily" | "once_daily" | "every_other_day"
│     | "weekly_n" (n回/週) | "custom"
├── timings: string[]（例：["08:00", "22:00"]）
├── durationDays: number | null（nullの場合は手動で次フェーズに進む）
├── targetBodyParts: BodyPartConfig[]
└── status: "pending" | "active" | "completed"

BodyPartConfig（部位設定）
├── bodyPartId: string（部位の識別子）
├── medication: string（薬剤名）
├── steroidRank: "strongest" | "very_strong" | "strong" | "medium" | "weak"
└── ftuAmount: number（自動算出 or 手動上書き）

ApplicationLog（塗布記録）
├── id: string (UUID)
├── phaseId: string
├── date: Date
├── timing: string（"morning" | "evening" | etc.）
├── appliedBodyParts: string[]（塗布した部位ID）
├── skipped: boolean（スキップした場合）
├── skipReason?: string（スキップ理由。任意）
└── createdAt: Date
```

### 4.2 ER図

```
TreatmentPlan  1──*  Phase  1──*  BodyPartConfig
                        │
                        │ 1
                        │
                        * 
                  ApplicationLog
```

## 5. 非機能要件

### 5.1 ユーザビリティ

- 塗布記録はワンタップで完了できること（操作の手間がリタイアの最大要因）
- 片手操作を前提としたUI（薬を塗った後の手で操作する可能性）
- 文字サイズは読みやすさを優先（小さすぎない）
- カラーユニバーサルデザインに配慮（色覚多様性対応）

### 5.2 パフォーマンス

- 初期表示 2秒以内
- 塗布記録のレスポンス 500ms以内

### 5.3 データ保護

- MVP段階ではデータはブラウザのローカルに保存
- 将来的にはアカウント連携・クラウド同期を検討
- 健康情報を扱うため、将来の拡張ではプライバシーポリシーの策定が必要

### 5.4 法的留意事項

- **医療機器・医療行為には該当しない設計とすること**
  - アプリは治療方針の決定・変更を行わない
  - あくまで「ユーザーが医師の指示を記録・管理するツール」
- 免責事項の表示：「本アプリは医療行為を行うものではありません。治療方針については必ず医師にご相談ください。」

## 6. MVP スコープ

### 6.1 MVP に含めるもの（Phase 1）

- 治療フェーズの登録・編集
- 今日のタスク表示＋塗布記録（ワンタップ）
- テーパリング進捗のプログレスバー
- ボディマップ（前面・背面、部位選択・薬剤紐付け）
- FTU目安表示（部位別の必要量）
- 塗布履歴のカレンダー表示
- ブラウザ通知によるリマインド

### 6.2 将来検討（Phase 2以降）

- 症状ログ（かゆみ・赤み・乾燥の記録と経過グラフ）
- チューブ残量管理＋補充リマインド
- 写真記録（部位の経過写真）
- 保湿剤の塗布管理（ステロイドと併用）
- 複数の治療計画管理（部位別に異なる計画）
- クラウド同期・アカウント機能
- 医師との共有機能（通院時にログを見せる）
- 多言語対応
- モバイルアプリ化（React Native）

## 8. 成功指標（KPI案）

アプリの効果を測定するための指標（将来的にトラッキングを検討）。

| 指標 | 目標 |
|------|------|
| 塗布完遂率（計画通りに塗った日数 / 計画日数） | 80%以上 |
| フェーズ完遂率（最終フェーズまで到達） | 60%以上 |
| 連続塗布日数（ストリーク） | 平均14日以上 |
| 再燃率（寛解後3ヶ月以内の再発） | 従来比で低減 |
